模块、包和NPM

一个大型的Node.js应用程序，通常是由多个模块组成的。模块是Node.js应用程序的基本单元。在Node.js中，文件和模块是一一对应的，也就是说，一个Node.js就是一个模块。文件可以是JavaScript代码，JSON文件或者编译过的C/C++扩展。

Node.js的包是在模块基础上的进一步抽象，其概念与C的函数库、Java的类库类似。

NPM（Node Package Manager，包管理器）是Node.js的包管理工具。

用Node.js开发Web应用程序系列教程（1）

本教程首先讲解如何搭建Node.js开发环境，然后演示如何用Node.js开发第一个小应用。

一、开发环境搭建

配置Node.js开发环境，需要几个步骤：

1.安装Node.js环境

到Node.js官方网站下载最新的Node.js安装包。Node.js支持Windows、Mac OS X、Linux和SunOS几种平台开发。在Windows环境下开发，我们只需要下载Windows installer即可。下载的时候将Node.js源代码一起下载下来（WebStorm下配置Node.js要用到）。

Windows平台下的Node.js安装非常简单，只需要点击安装文件，选中安装目录（例如d:\nodejs），然后一路Next下去就可以了。

安装完毕，打开Windows命令行控制台，输入node执行。如果没有错误提示，那就是说Node.js安装成功。

2.安装IDE

Node.js开发用NotePad++、Sublime Text和WebStorm都可以。这其中，WebStorm是功能最为强大的。搞Java开发的都知道有个比Eclipse、Netbeans更牛逼的Java IDE：Intellij IDEA。WebStorm和Intellij IDEA师出同门JetBrains，自然是继承了IDEA的优点，据称是最佳前端开发利器，我们就用它好了。

到JetBrains官网下载WebStorm的30天试用版。WebStorm个人版只要49美元，OSChina.net曾经搞了一个活动，中国区用户可以99元人民币就拿到。如果你盗版已经成习惯，那就百度一下吧，有破解的。

顺手也搜一下WebStorm的使用教程，多的是。BTW，WebStorm内置了Zen-coding，写html/css很快很快的。

3.配置WebStorm+Node.js环境

最新版的WebStorm内置了对Node.js的支持。我们只要稍加配置，就可以用WebStorm开发Node.js应用了。

打开WebStorm，在菜单栏上点击"File-->Settings"，打开WebStorm设置对话框。在对话框左边栏目中找到JavaScript节点，点击“Libraries“，查看是否安装了Node.js。如果安装有，就选中；否则就点击Download按钮下载。界面如下图所示。

注意：必须选中上图红色圆圈标注的选项，否则后面新建工程时候，Node.js源代码包含不到工程中。

在左边栏目中点击Node.js节点，将Node.js解释器设置为node.js安装目录下的node.exe（例如：d:\nodejs\node.exe）。然后安装Node.js核心模块源代码（下载Node.js时候，已经下载了）。配置完成后，界面如下图所示。

至此，基础开发环境搭建完毕。以后我们还将添加Express、Mongodb等的支持，这个后面再说。

查看Chrome浏览器缓存的方法

Google Chrome目前已经是性能最佳，对HTML5/CSS3等网页新技术支持最好的浏览器了，一般我浏览网站，习惯性地总是用Chrome。

最近一段时间，我经常用Chrome浏览国外教学视频网站。有时候看完好的视频后，总想把它从浏览器缓存中找出来保存起来，供教学参考。IE和FireFox下，很容易办到。没想到在Chrome下还费了点脑筋。

虽说Chrome下提供了一个命令chrome://cache，可以查看到保留下来的缓存，如下图1所示。

但是不幸的是，当你点击缓存文件，Chrome却并非打开缓存源文件，而是如图2所示的二进制编码文件。

这显然不是我想要的。那么，我还是跑到Chrome的缓存目录下去看看吧。

Windows 7操作系统中，Chrome的缓存目录是C:\Users{用户名}\AppData\Local\Google\Chrome\User Data\Default\Cache。打开缓存目录一看，得，什么都看不到，如图3所示。

这可如何是好呢？

Google了一下，还好，有个Chrome cache View工具，去下载绿色安装文件及中文语言包（BTW，nirsoft.net上面还有好多小工具哦）。当然，如果你只需要下载缓存的视频，还有个更厉害的工具Video cache View，IE/FF/Chrome通杀的！这个Chrome cache View还是不错，能看到很多缓存的参数，都可以用来分析、调试网页了，如图4所示。

OK，放心大胆用Chrome看视频吧。遇到有收藏价值的，就用Chrome Cache View或者Video Cache View保存起来。分析网页性能的时候，还可以用Chrome Cache View查看浏览器缓存的各项设置。
